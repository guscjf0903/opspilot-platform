장애 감지부터 안전 조치까지

  1. Dashboard에서 비정상 workload, warning event, Kafka lag를 확인합니다.
  2. Topology에서 Service, Pod, PVC, Node, Kafka Topic 관계를 따라갑니다.
  3. AI 분석에서 metric, event, log, rollout, lag 근거를 확인합니다.
  4. Cost 화면에서 request 대비 실제 사용량과 idle resource를 비교합니다.
  5. 운영 조치는 dry-run, diff, 승인, audit log를 거쳐 실행합니다.

시스템 구조

OpsPilot은 Kubernetes 안팎에서 실행될 수 있는 관측 도구입니다

Frontend

Vue 3 SPA가 Dashboard, Topology, Kafka, Cost, Incident 화면을 제공합니다.

Backend

Spring Boot API가 Kubernetes, Prometheus, Kafka, OpenCost, AWS Adapter를 조율합니다.

Data

PostgreSQL에는 사용자, snapshot, 분석 결과, 추천, audit log를 저장합니다.

External Systems

Kubernetes API, Redpanda/MSK, CloudWatch, Cost Explorer를 port/adapter로 분리합니다.

저비용 배포 전략

인프라는 포트폴리오를 보완하는 증거로만 사용합니다

  • 상시 공개는 GitHub Pages 정적 페이지로 처리해 AWS 비용을 만들지 않습니다.
  • AWS Budgets를 먼저 설정하고, EKS/ALB/RDS/MSK/NAT Gateway는 기본 비활성으로 둡니다.
  • 실제 EKS 데모는 Terraform으로 짧게 만들고 Helm으로 배포한 뒤 제거합니다.
  • Kafka는 기본 Redpanda로 비용을 줄이고, MSK는 필요할 때만 proof 모드로 검증합니다.
  • 외부 방문자는 read-only 화면만 보며 운영 action endpoint는 공개하지 않습니다.
